# /assess

Evaluate your current skills, knowledge level, and identify gaps in your development journey.

## Usage

```
/assess
/assess javascript
/assess react
/assess full-stack
```

## Description

Get a comprehensive assessment of your technical skills across different specializations and technologies.

## Assessment Features

### Self-Assessment Questionnaire
- Rate your proficiency across key technologies
- Identify strengths and areas for improvement
- Compare against role requirements
- Get personalized recommendations

### Assessment Areas

1. **Programming Languages**
   - JavaScript/TypeScript
   - Python
   - Java
   - Go
   - Rust

2. **Frontend Technologies**
   - HTML5 & CSS3
   - React, Vue, Angular
   - State management
   - Testing

3. **Backend Technologies**
   - Node.js, Python, Java
   - Database design
   - API design
   - Authentication

4. **DevOps & Infrastructure**
   - Docker & Kubernetes
   - CI/CD pipelines
   - Cloud platforms
   - Infrastructure as Code

5. **Soft Skills**
   - Communication
   - Teamwork
   - Problem-solving
   - Leadership

### Assessment Output

Get:
- **Proficiency Levels** for each skill area
- **Knowledge Gaps** that need attention
- **Strengths** to leverage
- **Improvement Plan** with specific recommendations
- **Benchmark** against industry standards
- **Time Estimate** to reach proficiency

## Assessment Methods

### Quick Assessment (15 minutes)
- Self-reported skill levels
- Basic knowledge checks
- Quick evaluation

### Comprehensive Assessment (1-2 hours)
- In-depth questionnaire
- Technical questions
- Code review exercise
- Scenario-based problems

### Continuous Assessment
- Track progress over time
- Re-assess every 3 months
- See improvement trends
- Adjust learning paths

## How to Use Results

1. **Review** your assessment results
2. **Identify** top 3 areas for improvement
3. **Use /plan** to create targeted learning path
4. **Use /projects** to find practice projects
5. **Re-assess** in 3 months to measure progress

## Tips for Honest Assessment

- Be realistic about your skills
- Don't overestimate knowledge
- Consider actual experience (not just reading)
- Assess hands-on capability, not just theory
- Focus on what you can do, not what you know

## Next Steps

After assessment:
- Use `/plan` to create personalized learning path
- Join study groups with similar assessments
- Find mentors who excel in your weak areas
- Focus 80% of effort on improving gaps
- Build projects to solidify learning
